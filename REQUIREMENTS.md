# pymanager 项目需求文档

> 本文档用于记录 pymanager 的现有功能、待实现功能建议以及同类优秀功能参考。

## 一、项目定位

pymanager 是一个基于 Tkinter 的 Python 脚本管理工具，帮助用户：
- 保存和管理本地 Python 脚本
- 快速运行脚本
- 管理脚本依赖

## 二、现有功能

| 模块 | 功能 | 实现方式简述 | 备注 |
|------|------|-------------|------|
| 添加脚本 | 从文件添加脚本到管理库 | 文件对话框选择 .py 文件，复制到 `data/` 目录；支持拖拽添加 | `add_script.py` / `drag_drop.py` |
| 删除脚本 | 删除选中的脚本 | 从 `data/` 目录删除对应文件并刷新列表 | `delete_selected.py` |
| 重命名脚本 | 重命名选中脚本 | 重命名 `data/` 中的文件并刷新列表 | `rename_selected.py` |
| 编辑脚本 | 编辑脚本内容 | 读取文件内容到 Text 控件，保存时写回文件 | `edit_content.py` |
| 运行脚本 | 运行选中的脚本 | `subprocess.Popen` 启动子进程 + `threading.Thread` 后台读取输出，实时显示到 Text 控件 | `run_selected.py`；⚠️ UI 更新未使用 `root.after`，存在线程安全风险 |
| 分组管理 | 按分组组织脚本，支持新建/删除分组 | 基于 `data/` 子文件夹结构，子文件夹名即分组名；根目录脚本归入"默认分组" | `group_manager.py`；`save_groups` 已为空操作 |
| 依赖检查 | 自动检测和安装脚本依赖 | ① `ast.parse` 解析脚本 import 语句 → ② 过滤标准库（`sys.stdlib_module_names` / `importlib`）和已安装包 → ③ `pip install` 多源容错安装（清华源→阿里云源→中科大源→PyPI） | `dependencies.py` / `check_deps.py`；后台线程运行，UI 更新使用 `root.after` |
| 自动更新 | 检查并下载 GitHub 最新版本 | 调用 GitHub Releases API，使用加密存储的 Token 认证 | `updater.py` / `token_crypto.py` |
| 拖拽支持 | 拖拽 .py 文件自动添加 | `tkinterdnd2` 库实现拖拽事件监听 | `drag_drop.py`；启动时检查 `tkinterdnd2` 是否已安装 |
| 右键菜单 | 移动脚本到其他分组 | 右键弹出菜单，选择目标分组后移动文件到对应子文件夹 | 集成在 `main.pyw` |
| 安全存储 | 支持敏感信息（如 API Token）的加密存储 | Windows DPAPI（`CryptProtectData` / `CryptUnprotectData`），base64 编码写入 `config/api_token.enc` | `token_crypto.py`；仅限 Windows 平台 |
| 配置管理 | 集中化管理应用路径配置 | `config.py` 导出路径常量（`BASE_DIR`、`DATA_DIR`、`CONFIG_FILE`）；`storage.py` 提供 pickle 序列化但当前主流程未使用 | `config.py` / `storage.py`；⚠️ `storage.py` 处于半废弃状态 |

## 五、推荐优先实现功能

基于开发成本和用户价值，建议按以下优先级实现：

### 第一阶段（稳定性 + 易用性）

1. **配置文件 JSON 化** - P0 级基础任务，后续元数据、标签系统都依赖灵活的数据存储格式
2. **脚本搜索/过滤** - 提升日常使用效率
3. **快捷键支持** - 提升操作效率
4. **运行历史 + 统一输出面板** - 记录脚本执行记录，合并展示脚本输出和日志
5. **依赖安装确认** - 避免用户不知情安装包，安全性提升

### 第二阶段（增强功能）

6. **主题配色文件支持** - 先实现 `color.ini` 配色方案，降低深色模式开发成本
7. **批量操作** - 提升批量管理效率
8. **脚本收藏/置顶** - 方便常用脚本
9. **脚本图标** - 便于识别
10. **脚本修改历史** - 自动备份，防止误修改
11. **依赖版本显示** - 便于依赖管理
12. **运行前参数输入框** - 提升运行灵活性

### 第三阶段（长期）

13. **深色模式** - 在主题配色文件基础上实现
14. **脚本标签系统 + 自动标签** - 超越文件夹的分类方式
15. **#pqr 脚本头注释** - 让脚本自描述
16. **脚本模板** - 快速创建新脚本
17. **定时运行（简单版）** - 自动化脚本执行
18. **云同步** - 跨设备同步配置

## 六、技术债务

| 问题 | 描述 | 建议方案 | 优先级 |
|------|------|---------|--------|
| 数据存储 | 目前使用 pickle，难以手动编辑且存在安全风险；`storage.py` 处于半废弃状态（主流程已改用目录扫描） | **P0:** 迁移至 JSON，设计版本号字段以便未来扩展；清理 `storage.py` 废弃代码 | P0 |
| 日志系统 | 目前混用 print 和 logger | 统一使用 `logging`，支持将日志输出到 GUI 的统一输出面板 | P1 |
| 异常处理 | 部分异常处理过于宽泛 | 细化异常类型 | P1 |
| 代码耦合 | 部分模块间耦合较紧 | 继续按单一职责拆分 | P2 |
| 测试覆盖 | 缺少单元测试 | 添加基础测试 | P2 |
| 路径硬编码 | 目前存储了脚本的绝对路径，可移植性差 | 统一替换为相对路径（基于应用目录或用户配置的脚本根目录） | P1 |
| 线程安全 | `run_selected.py` 在后台线程中直接操作 Tkinter 控件，未使用 `root.after` 调度回主线程 | 所有耗时操作的 UI 更新必须通过 `root.after(0, callback)` 调度到主线程 | P1 |
| UI 框架 | 原生 Tkinter 样式老旧，自定义组件开发效率低 | 评估引入 `ttkbootstrap` 或 `customtkinter` 提升颜值和开发效率 | P2 |
| 进程管理 | 运行脚本时缺少"停止运行"按钮，无法终止长时间运行的子进程 | 为每个运行中的子进程保存 `process` 引用，提供"停止"按钮调用 `process.terminate()` | P1 |

## 七、非功能性需求

| 需求 | 描述 | 目标 |
|------|------|------|
| 启动速度 | 随着脚本数量增加，搜索和加载速度不能明显下降 | 100 个脚本内启动时间 < 2 秒 |
| 安全性 | 运行用户脚本时需考虑安全隔离 | 提供依赖安装确认；避免 `shell=True` 防止命令注入（已实现） |
| 兼容性 | 明确支持的 Python 版本范围 | Python 3.8+；Windows 为主要支持平台（Token 加密依赖 Windows DPAPI） |
| 可移植性 | 配置和脚本数据应能在不同机器间迁移 | 路径使用相对路径，配置使用 JSON 格式 |
