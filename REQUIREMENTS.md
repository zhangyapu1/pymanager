# pymanager 项目需求文档

> 本文档用于记录 pymanager 的现有功能、待实现功能建议以及同类优秀功能参考。

## 一、项目定位

pymanager 是一个基于 Tkinter 的 Python 脚本管理工具，帮助用户：
- 保存和管理本地 Python 脚本
- 快速运行脚本
- 管理脚本依赖

## 二、现有功能

| 模块 | 功能 |
|------|------|
| 添加脚本 | 从文件添加脚本到管理库 |
| 删除脚本 | 删除选中的脚本 |
| 重命名脚本 | 重命名选中脚本 |
| 编辑脚本 | 编辑脚本内容 |
| 运行脚本 | 运行选中的脚本 |
| 分组管理 | 按分组组织脚本，支持新建/删除分组 |
| 依赖检查 | 自动检测和安装脚本依赖 |
| 自动更新 | 检查并下载 GitHub 最新版本 |
| 拖拽支持 | 拖拽 .py 文件自动添加 |
| 右键菜单 | 移动脚本到其他分组 |

## 三、待实现功能

### 3.1 核心功能增强

#### P0 - 高优先级

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **脚本搜索/过滤** | 在列表上方添加搜索框，支持按名称快速过滤脚本 | 低 |
| **脚本收藏/置顶** | 支持将常用脚本标记为收藏，优先显示 | 低 |
| **脚本图标** | 支持为脚本设置自定义图标，便于识别 | 中 |

| 功能 | 描述 | 复杂度 |
|------|------|--------|
| **批量操作** | 支持多选脚本，批量删除/移动/导出 | 中 |

| **脚本搜索增强** | 支持按标签、脚本描述、更新时间等条件组合搜索 | 高 |
| **最近运行** | 记录并显示最近运行的脚本列表 | 低 |

## 五、推荐优先实现功能

基于开发成本和用户价值，建议按以下优先级实现：

### 第一阶段（1-2周）

1. **脚本搜索/过滤** - 提升日常使用效率
2. **快捷键支持** - 提升操作效率
3. **运行历史** - 记录脚本执行记录
4. **日志查看器** - 便于问题排查

### 第二阶段（1个月）

5. **深色模式** - 改善用户体验
6. **批量操作** - 提升批量管理效率
7. **脚本收藏/置顶** - 方便常用脚本
8. **依赖版本显示** - 便于依赖管理

### 第三阶段（长期）

9. **脚本标签系统** - 超越文件夹的分类方式
10. **脚本模板** - 快速创建新脚本
11. **定时运行** - 自动化脚本执行
12. **云同步** - 跨设备同步配置

## 六、技术债务

| 问题 | 描述 | 建议 |
|------|------|------|
| 配置文件格式 | 目前使用 pickle，建议迁移到 JSON | 降低依赖，提高可移植性 |
| 日志系统 | 目前混用 print 和 logger | 统一使用 logging |
| 异常处理 | 部分异常处理过于宽泛 | 细化异常类型 |
| 代码耦合 | 部分模块间耦合较紧 | 继续按单一职责拆分 |
| 测试覆盖 | 缺少单元测试 | 添加基础测试 |
